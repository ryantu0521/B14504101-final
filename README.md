(1)程式原理與功能:這個程式的功能為能基於使用者給的酸鹼濃度以及含量畫出對應的滴定曲線，酸和鹼可包含多元酸鹼以及強弱酸鹼，並有給予一些常用的酸鹼可供選擇，也可由使用者給予一個自定義的酸鹼，最後畫出滴定曲線以及到達當量點所用的體積且指出最適合此次滴定的指示劑。原理部分主要使用的方程式有電荷平衡$$ [H^+] + [Cations]{base} = [OH^-] + [Anions]{acid} $$ 平均電荷$$ \bar{Q}{acid} = \frac{\sum{i=1}^{n} i \cdot [H_{n-i}A^{i-}]}{C_{total}} $$ 並推導出體積公式$$ V_t = V_0 \frac{C_0 \bar{Q}{acid} - \Delta{water}}{C_t \bar{Q}{base} + \Delta{water}} $$ 計算滴定體積。
(2)使用方式:先選擇被滴定的是酸或鹼，再選擇所選用的酸鹼為何，若選擇自定義酸鹼則須給出pKa或是pKb，若是多元酸鹼則將每個值用逗號隔開，最後輸入被滴定溶液的濃度以及體積和滴定溶液的濃度，即可得到滴定曲線,到達當量點體積以及最適指示劑。
(3)程式架構:參數設定 (輸入濃度、體積與物質) - 邏輯驗證 (檢查弱酸弱鹼無效組合) - 數值模擬 (執行電荷平衡逆向求解) - 結果分析 (計算當量點與推薦指示劑) - 圖表輸出 (繪製 pH 曲線與標記)
(4)開發過程:1.過程中原本是只要輸入隨便的酸鹼就能執行，但若遇到弱酸滴定弱鹼時，會因為缺乏明顯的跳躍導致無法準確判斷當量點，因此最後決定讓程式不提供弱滴定弱的功能來避免問題產生。2.若使用正常濃度計算，再計算多質子酸鹼時會用到大量if-else導致程式變得很複雜，因此最後引進電荷平衡改善問題。
(5)參考資料來源:Google Gemini，主要為討論改進部份以及取得酸鹼的pKa,pKb值和指示劑的變色範圍
(6)1.加入顯示最適合滴定的指示劑讓使用者能在拿到未知濃度和容量準備進行實驗之前得知需使用何種指示劑。2.當初原本是想做能從列表中選出要使用的酸鹼，但後來決定加入自訂酸鹼的部分能在當列表中沒有時藉由KaKb也能畫出正確的滴定曲線增加實用性。
